Handling the iOS keyboard backspace button

How to tame the sonofabitch.


I recently had to create a numerical code input which consisted of six separate UITextField inputs. To make this it even remotely usable, I had to attach editing events to bump the cursor onto the next input when a number was typed. The same had to be done for the backspace but it wasn’t as easy as latching onto the Editing Changed IBAction.

After quite a bit of searching, a user named Mysiaq on StackOverflow posted this underrated snippet. He mentions overriding the UITextField deleteBackward method to trigger a backspace press event. Brilliant. So I got to work fitting it into my code.

To understand it, some digging around is required first. First of all, I used Xcode’s excellent “Jump to definition” feature. Take a look at the UITextField’s interface and you’ll see it implements the UITextInput protocol. This protocol prototypes (the correct verb?) the -(void) deleteBackward responder which is what were going to latch our callback onto. Let’s start by creating a new class in Xcode.
